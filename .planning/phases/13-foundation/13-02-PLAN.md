Create `get-shit-done/bin/lib/dispatcher/stderr-bridge.js` as a CommonJS module.

**Imports:**
- `redactSecrets` from `./sanitize.js` (the ONLY external import — sibling file, clean relative path)

**Exports:**
- `STDERR_TAIL_LINES = 7` — within the 5-10 range per user discretion
- `buildRecoveryPayload(runnerResult, command)` — main function

**buildRecoveryPayload implementation (per locked decisions):**
1. If `runnerResult` is null/undefined or `runnerResult.code === 0` → return `null`
2. Extract raw stderr: `runnerResult.stderr || ''`
3. Tail truncation: split on `\n`, slice last `STDERR_TAIL_LINES` lines, rejoin and trim
4. Redact stderr hint: `const { redacted: stderrHint } = redactSecrets(tail)`
5. Redact command: `const { redacted: redactedCommand } = redactSecrets(command || '')`
6. Return `{ exitCode: runnerResult.code, stderrHint, command: redactedCommand }`

**Critical design constraints (from CONTEXT.md locked decisions):**
- Minimal payload: exitCode, stderrHint, command ONLY — no error category, no suggested action
- Returns null on success (code === 0) — caller checks for null
- This is a PURE function — no I/O, no side effects, no file writes
- Phase 14 adds recovery routing on top; this module just surfaces the data

**Anti-patterns to avoid (from research):**
- Do NOT redact after constructing the payload object — redact raw strings first
- Do NOT import from `./index.js` — only `./sanitize.js`
- Do NOT add error categories or recovery suggestions (deferred to Phase 14)
