Create `get-shit-done/bin/lib/session/store.js` as a CommonJS module (project convention — all existing modules use `require`/`module.exports`).

**Imports:**
- `node:fs` — readFileSync, writeFileSync, renameSync, chmodSync, statSync
- `node:path` — dirname, join
- `redactSecrets` from `../dispatcher/sanitize.js` (the ONLY external import)

**Exports:**
- `RING_SIZE = 3` — hardcoded constant (per SESS-01 and discretion decision)
- `appendRecord(filePath, { command, exitCode, stderrSnippet })` — main write function
- `readRecords(filePath)` — read helper (thin wrapper for tests and future use)

**appendRecord implementation:**
1. Read existing entries via `readRecords(filePath)` (returns `[]` on missing/corrupt file)
2. Redact command: `const { redacted: redactedCommand } = redactSecrets(command || '')`
3. Redact stderrSnippet: `const { redacted: redactedSnippet } = redactSecrets(stderrSnippet || '')`
4. Build record: `{ command: redactedCommand, exitCode, stderrSnippet: redactedSnippet, timestamp: new Date().toISOString() }`
5. Push record, slice to last `RING_SIZE` entries if over limit
6. Atomic write:
   - Temp path: `path.join(path.dirname(filePath), \`.session-${process.pid}-${Date.now()}.tmp\`)`
   - `fs.writeFileSync(tmp, JSON.stringify(entries, null, 2), 'utf8')`
   - `fs.renameSync(tmp, filePath)`
   - `fs.chmodSync(filePath, 0o600)`

**readRecords implementation:**
1. Try `JSON.parse(fs.readFileSync(filePath, 'utf8'))`
2. Catch any error (ENOENT, SyntaxError) → return `[]`
3. If parsed value is not an array → return `[]`

**Critical anti-patterns to avoid (from research):**
- Do NOT use `os.tmpdir()` for temp file — must be same dir as target for atomic rename
- Do NOT rely on `writeFileSync` mode option for permissions — use `chmodSync` after rename
- Do NOT catch errors silently in appendRecord — let them propagate (Phase 14 wraps)
- Do NOT store stdout in the record (out of scope per requirements)
- Redact BEFORE constructing the record, not after

Run all tests: `node --test get-shit-done/bin/lib/session/__tests__/store.test.js` — all must pass (GREEN).
