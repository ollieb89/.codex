  <action>
Create `get-shit-done/bin/lib/dispatcher/commands.js` as a pure CommonJS module with `'use strict'` and no dependencies. Export these named constants:

1. **BLOCKED_COMMANDS** — `new Set(['sudo', 'chown', 'chmod', 'mkfs', 'dd'])` (from sanitize.js line 4)
2. **ALLOWLIST** — `new Set(['git', 'npm', 'yarn', 'pnpm', 'node', 'mkdir', 'cp', 'mv', 'ls', 'cat', 'touch', 'echo', 'grep', 'rg'])` (from sanitize.js lines 5-20)
3. **GRAY_COMMANDS** — Array of objects with `prefix` (string) and `reason` (string) fields:
   - `{ prefix: 'git push', reason: 'Pushes to remote' }`
   - `{ prefix: 'npm publish', reason: 'Publishes to registry' }`
   - `{ prefix: 'pnpm publish', reason: 'Publishes to registry' }`
   - `{ prefix: 'yarn publish', reason: 'Publishes to registry' }`
   Per user decision: replaces the old `Set` approach with prefix-based matching.
4. **DESTRUCTIVE_HIGHLIGHT_TERMS** — `['rm', 'truncate', 'drop', 'overwrite', '--force', '-rf']` (from preview.js line 9)
5. **MUTATING_PATTERN** — Build a `RegExp` from the canonical term list `['rm', 'mv', 'cp', 'push', 'write', 'append', 'truncate', 'npm publish', 'git push']`. For each term: escape regex special chars, replace whitespace with `\\s+`, wrap in `\\b` anchors. Join with `|`, compile with `'i'` flag. Store the terms array as `MUTATING_TERMS` (not exported) for maintainability.

Also export a helper function **matchGray(command)** that sanitize.js will call:
- Trim and lowercase the command string
- Return the first `GRAY_COMMANDS` entry where `lower === entry.prefix` OR `lower.startsWith(entry.prefix + ' ')`
- Return `null` if no match
The trailing-space guard prevents prefix collisions (e.g., `git push-mirror` must NOT match).

Use `module.exports = { BLOCKED_COMMANDS, ALLOWLIST, GRAY_COMMANDS, DESTRUCTIVE_HIGHLIGHT_TERMS, MUTATING_PATTERN, matchGray }`.

Do NOT add any logic beyond constant definitions and the matchGray helper. No side effects.
